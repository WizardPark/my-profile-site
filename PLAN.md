# WizardPark 프로필 웹사이트 - 사용자 수정 가이드

## 📋 개요

이 웹사이트는 **`config.js` 파일 하나만 수정**하면 전체 내용이 반영되는 구조로 설계되었습니다.  
각 필드는 한국어 주석으로 설명되어 있으므로, 필요한 부분만 수정하면 됩니다.

---

## 📁 파일 구조

```
my-profile-site/
├── index.html          # 웹페이지 구조 (수정 불필요)
├── config.js           # ★ 여기만 수정하세요 (이름, 소개, 스킬, 프로젝트, 연락처)
├── main.js             # 렌더링 로직 (수정 불필요)
├── assets/
│   └── images/         # 프로젝트 이미지 폴더 (이미지 파일 저장)
└── PLAN.md             # 이 파일
```

---

## ✏️ 수정 방법

### 1️⃣ 기본 정보 수정

`config.js` 파일을 텍스트 에디터로 열어서 다음 항목들을 수정합니다:

```js
const PROFILE = {
  // 이름
  name: "WizardPark",  // ← 여기를 수정

  // 직업/역할
  role: "Firmware Developer",  // ← 여기를 수정

  // 자기소개 (배열의 각 항목이 한 줄)
  intro: [
    "첫 번째 줄",  // ← 수정
    "두 번째 줄",  // ← 수정
    "세 번째 줄"   // ← 수정
  ],
  ...
};
```

---

### 2️⃣ 기술 스택 수정

기술 스택을 추가, 삭제, 수정할 수 있습니다:

```js
skills: [
  // 항목 추가 예시
  { name: "C Language", level: "중급", category: "Languages" },
  { name: "Python", level: "중급", category: "Languages" },
  // ... 다른 항목들
]
```

**필드 설명:**
- `name`: 기술명 (예: "C Language", "STM32")
- `level`: 숙련도 (예: "초급", "중급", "상급")
- `category`: 카테고리 (예: "Languages", "MCU/Embedded", "GUI", "EDA/PCB")

---

### 3️⃣ 프로젝트/포트폴리오 수정

각 프로젝트는 다음 필드를 포함합니다:

```js
projects: [
  {
    title: "프로젝트 제목",           // ← 프로젝트명 수정
    period: "2023.01 ~ 2023.06",    // ← 개발 기간 수정
    purpose: "목적 설명",            // ← 개발 목적 수정
    desc: "상세 설명",               // ← 상세 설명 수정
    image: "이미지 경로",            // ← 이미지 경로 수정 (아래 참고)
    tags: ["태그1", "태그2"]         // ← 기술 태그 수정
  },
  ...
]
```

**이미지 교체 방법:**

1. 프로젝트 이미지를 준비합니다 (JPG, PNG 등)
2. `assets/images/` 폴더에 이미지 파일을 저장합니다
   ```
   my-profile-site/
   └── assets/
       └── images/
           ├── stm32_project.jpg      # ← 새 이미지
           ├── avr_motor.jpg          # ← 새 이미지
           └── ...
   ```
3. `config.js`의 `image` 필드를 수정합니다
   ```js
   {
     title: "STM32 프로젝트",
     image: "assets/images/stm32_project.jpg",  // ← 파일명 변경
     ...
   }
   ```
4. 브라우저를 새로고침하면 이미지가 바로 반영됩니다.

**외부 URL 사용도 가능:**

이미지를 인터넷에서 직접 가져올 수도 있습니다:
```js
image: "https://example.com/my-image.jpg"  // ← 외부 URL도 가능
```

### 3️⃣ Career (경력) 수정

경력 순서는 **대학교 → 대학원 → 업체1 → 업체2 → 업체3** 순으로 입력합니다.

```js
career: [
  {
    type: "education",        // "education" (학교) 또는 "work" (회사)
    org: "OO대학교",          // ← 기관명 수정
    role: "전자공학과 학사",   // ← 전공/직위 수정
    startDate: "2015.03",     // ← 시작일 수정 (YYYY.MM 형식)
    endDate: "2019.02",       // ← 종료일 수정 (YYYY.MM 형식 또는 "현재")
    exitReason: "졸업",       // ← 사유 선택: 졸업 / 재직중 / 자진퇴사 / 권고사직 / 계약만료
    desc: "설명"              // ← 추가 설명 (생략 가능)
  },
  // ... 추가 항목
]
```

**자동 계산 기능:**
- `startDate`와 `endDate`를 입력하면 **근무일수가 자동 계산**됩니다 (예: "3년 11개월 5일")
- `endDate: "현재"` 입력 시 → 오늘 날짜 기준으로 실시간 계산

**퇴사 사유 배지 색상:**
| 사유 | 배지색 | 효과 |
|------|--------|------|
| 졸업 | 🟢 초록 | - |
| 재직중 | 🔵 하늘 | 깜빡임 점 |
| 자진퇴사 | 🟡 노랑 | - |
| 권고사직 | 🔴 빨강 | - |
| 계약만료 | ⚫ 회색 | - |

---

### 3-1️⃣ Achievements (업적) 수정

논문, 특허, 수상 기록 등을 입력합니다.

```js
achievements: [
  {
    type: "paper",      // "paper" (논문) | "patent" (특허) | "award" (수상)
    title: "논문 제목",  // ← 제목 수정
    date: "2021.06",    // ← 날짜 수정 (YYYY.MM 형식)
    issuer: "OO학회",   // ← 발행/수여 기관 수정
    desc: "간단한 설명" // ← 설명 수정
  },
  // ... 추가 항목
]
```

**타입별 아이콘 및 배지색:**
| 타입 | 아이콘 | 배지색 |
|------|--------|--------|
| paper (논문) | 📄 | 파랑 |
| patent (특허) | 🔧 | 보라 |
| award (수상) | 🏆 | 금색 |

## 💡 추가 팁

### 배열에서 항목 추가하기

```js
// 기술 스택에 새 항목 추가
skills: [
  { name: "C Language", level: "중급", category: "Languages" },
  { name: "Rust", level: "초급", category: "Languages" },  // ← 새 항목 (쉼표 주의!)
  { name: "Python", level: "중급", category: "Languages" }
]
```

### 배열에서 항목 삭제하기

```js
// 특정 항목을 삭제하려면 전체 줄을 지웁니다
skills: [
  { name: "C Language", level: "중급", category: "Languages" },
  // { name: "Rust", level: "초급", category: "Languages" },  // ← 삭제됨
  { name: "Python", level: "중급", category: "Languages" }
]
```

### JSON 문법 주의사항

- **쉼표**: 배열 항목 끝에 쉼표 `,`가 있어야 합니다 (마지막 항목 제외)
- **따옴표**: 문자열은 큰따옴표 `"` 또는 작은따옴표 `'`로 감싸야 합니다
- **중괄호/대괄호**: 열고 닫은 개수가 맞아야 합니다

## ❓ 문제 해결

### 이미지가 안 보여요
- `assets/images/` 폴더가 있는지 확인하세요
- 이미지 파일명이 `config.js`의 경로와 정확히 일치하는지 확인하세요
- 특수문자나 공백이 파일명에 없는지 확인하세요

### 글자가 깨져요
- 파일을 **UTF-8** 인코딩으로 저장했는지 확인하세요 (VS Code 기본값)

### 프로젝트가 화면에 안 나타나요
- `config.js`에서 `projects` 배열에 올바른 문법이 사용되었는지 확인하세요
- 중괄호 `{}` 와 대괄호 `[]`가 정확히 닫혀있는지 확인하세요

## 📞 요약

**`config.js` 파일의 다음 부분만 수정하면 됩니다:**

- ✏️ `name` — 이름
- ✏️ `role` — 직업/역할
- ✏️ `intro` — 자기소개
- ✏️ `career[]` — 경력 (기관명, 기간, 퇴사 사유, 설명 등)
- ✏️ `skills` — 기술 스택
- ✏️ `achievements[]` — 업적 (논문, 특허, 수상 등)
- ✏️ `projects` — 프로젝트/포트폴리오 (제목, 기간, 목적, 설명, 이미지, 태그)
- ✏️ `contact.email` — 이메일
- ✏️ `contact.phone` — 전화번호

**나머지 파일은 수정할 필요가 없습니다!**
